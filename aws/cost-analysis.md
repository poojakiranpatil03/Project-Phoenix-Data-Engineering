# Service-wise Cost Analysis

## Amazon S3

### Purpose

Stores incoming source files before Snowflake ingests them.

### Pricing

Amazon S3 charges for:

- Storage used (GB per month)
- PUT requests
- GET requests
- Data transfer (when applicable)

### Project Phoenix Usage

- Small CSV files
- Few uploads per day
- Minimal storage consumption

Estimated Monthly Cost

Approximately ₹0–₹10 while using the AWS Free Tier.

### Cost Optimization

- Delete obsolete files periodically.
- Store only source data required for learning.
- Organize files using prefixes instead of creating multiple buckets.

---

## Amazon SNS

### Purpose

Broadcast file arrival notifications generated by Amazon S3.

### Pricing

Charged based on:

- Number of published messages
- Number of notification deliveries

### Project Phoenix Usage

- One notification generated per uploaded file.
- Very low message volume.

Estimated Monthly Cost

Typically ₹0 within the AWS Free Tier.

### Cost Optimization

- Publish only meaningful events.
- Filter unnecessary notifications using prefixes and suffixes.

---

## Amazon SQS

### Purpose

Provide reliable message delivery between Amazon SNS and Snowflake.

### Pricing

Charged based on:

- Number of API requests

### Project Phoenix Usage

Snowflake consumes messages automatically after file uploads.

Estimated Monthly Cost

Typically ₹0 within the AWS Free Tier.

### Cost Optimization

- Avoid unnecessary polling.
- Delete messages after successful processing (handled automatically by Snowflake).

---

## IAM

### Purpose

Secure communication between Snowflake and AWS.

### Pricing

IAM has no additional charges.

Estimated Monthly Cost

₹0

---

## S3 Event Notifications

### Purpose

Automatically notify Amazon SNS whenever a new file is uploaded.

Pricing

No additional charge.

Estimated Monthly Cost

₹0

# Snowflake Cost Considerations

Although AWS costs remain minimal, Snowflake compute consumption should also be considered.

### Warehouse

Warehouse | RETAIL_WH
Warehouse Size | X-Small
Auto Suspend | 60 Seconds
Auto Resume | Enabled

These settings minimize compute costs by ensuring that the warehouse runs only when required.

### Snowpipe

Snowpipe is a serverless service.

Charges are based on:

- Number of files processed
- Compute resources consumed during ingestion

For Project Phoenix, the ingestion volume is extremely small, making Snowpipe costs negligible.

---

# Cost Optimization Strategies

Project Phoenix follows several cloud cost optimization best practices:

✅ Use an X-Small Warehouse

✅ Enable AUTO_SUSPEND

✅ Enable AUTO_RESUME

✅ Process only incremental files

✅ Use Event-Driven Snowpipe instead of scheduled polling

✅ Store only small learning datasets

✅ Organize S3 data using prefixes

✅ Monitor Snowflake usage regularly

# Key Learnings

Building cloud-native data pipelines does not necessarily require expensive infrastructure.

By combining:

- Amazon S3
- Amazon SNS
- Amazon SQS
- Snowpipe
- Auto Suspend Warehouses

it is possible to design a scalable, event-driven architecture while keeping operational costs minimal.

Understanding cloud pricing is just as important as understanding cloud architecture, as cost optimization is a key responsibility of modern data engineers and cloud architects.
